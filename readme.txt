Все начинается от файла AuthPage.
В нем 3 роута 
    - /entercode
    - /register
    - /

1) /
    Открывается Auth и проверяется кукис, если все есть - AuthMe,
    если нет, то проходим авторизацию.
2) register
    Берем эмейл и вызываем getCode
3) entercode
    Ввод кода одинаков для регистрации и авторизации, в стейте есть переменная isLogging,
    если true, запрос логина, false - запрос регистрации

    В обеих случаях при правильном вводе генерируем сид и храним в кукис.


-----------------------------------------------------------------------------
Запросы на эндпоинты делается в файлах authApi, excursionApi
Вся логика авторизации и получения екскурсии хранится в authReducer
Action-ы actionCreator-ы в actions.js

getCode, checkCode - регистрация
getRestoreCode, checkRestoreCode - авторизация

Большая часть логики происходит в AuthMe
Сначала происходит авторизация, и если сервер ответит ок, делаем запрос для подключения к стриму, 
если опять ок, вызывается alreadyOk.

В alreadyOk уже по комментариям можно понять где какой пункт. Просто в нужных местах надо добавить редиректы на 
стрим, оплату, или показать экран ожидания.

После успешного авторизации попадаем в StartPage, там есть кнопка Подключится. При нажатии работает joinMe из excoursionReducer,
в нем тоже работает alreadyOk.
